2) Root Cause Analysis

RC-1: Gateway order-create limiter became the dominant bottleneck.
- Current sandbox gateway default:
  - GATEWAY_ORDER_CREATE_RATE_LIMIT_MAX_REQUESTS=20
  - GATEWAY_ORDER_CREATE_RATE_LIMIT_WINDOW_SECONDS=60
- Source:
  - infra/docker/docker-compose.sandbox.yml
  - gateway/app/core/config.py
  - gateway/app/core/proxy.py

RC-2: The S1 test model triggers create per iteration.
- In unique_orders path, each loop tries create -> pay -> approve/reject.
- At 500 VUs, create QPS greatly exceeded gateway policy, so 429 flood happened before order-service concurrency behavior could be observed.

RC-3: Test objective and protection policy were mixed in one run.
- Goal of 7.2 is order-service checkout concurrency behavior.
- Actual S1 mostly measured gateway protection behavior.

  - SANDBOX_ORDER_CREATE_RATE_LIMIT_WINDOW_SECONDS=60

Lane 2: Gateway protection lane (separate report)
